# Abstract Algebra

## Overview

Abstract algebra studies **structure** rather than numbers — sets equipped with
operations that obey a handful of axioms. Strip arithmetic down to "a set, an operation,
and some rules" and the same skeleton reappears in clock arithmetic, symmetries,
matrices, and polynomials. The payoff for computing is concrete: the **groups, rings, and
fields** here are the machinery behind [cryptography](../security/encryption.md) and
[post-quantum schemes](../security/post_quantum_crypto.md), error-correcting codes and
[information theory](information_theory.md), hashing, and fast arithmetic. It is the
natural continuation of the number theory and modular arithmetic in
[discrete math](discrete_math.md).

```
The structure ladder — each adds operations/axioms:

  Group   one operation, invertible            (symmetry, rotation)
  Ring    add + multiply (no division needed)  (integers, polynomials)
  Field   ring where every nonzero element     (rationals, reals, GF(p))
          has a multiplicative inverse
```

## Groups

A **group** is a set `G` with one operation `·` satisfying four axioms:

```
1. Closure        a·b ∈ G                      stays inside the set
2. Associativity  (a·b)·c = a·(b·c)
3. Identity        ∃ e:  e·a = a·e = a          (0 for +, 1 for ×)
4. Inverse         ∃ a⁻¹: a·a⁻¹ = e             every element undoable

Abelian (commutative) group also has:  a·b = b·a
```

Examples: integers under addition `(ℤ, +)`; nonzero rationals under multiplication;
rotations of a square under composition (non-abelian). The **order** of a group is its
size; the **order of an element** `a` is the smallest `k` with `aᵏ = e`.

## Cyclic groups and generators

A group is **cyclic** if a single element `g` (a **generator**) produces everything by
repetition: `G = {g⁰, g¹, g², …}`.

```
(ℤ₆, +) is cyclic, generated by 1:   1,2,3,4,5,0
Multiplicative group mod 7 is cyclic, generator 3:
   3¹=3, 3²=2, 3³=6, 3⁴=4, 3⁵=5, 3⁶=1   → hits all of {1..6}
```

Generators are exactly what public-key crypto exploits: `gˣ mod p` is cheap to compute
but inverting it (the **discrete logarithm** — recovering `x`) is believed hard. That gap
is Diffie–Hellman and ElGamal; see [encryption](../security/encryption.md).

## Modular arithmetic and the integers mod n

`ℤₙ = {0, 1, …, n−1}` with arithmetic wrapped around `n` is the most-used finite
structure in computing.

```
  (ℤₙ, +)         always a group
  (ℤₙ*, ×)        the units (elements coprime to n) form a group
  ℤₚ for prime p  every nonzero element is invertible  → a FIELD
```

Inverses come from the [extended Euclidean algorithm](discrete_math.md), and
**Fermat's little theorem** (`aᵖ⁻¹ ≡ 1 mod p`) and Euler's theorem power RSA.

## Rings and fields

```
RING   (R, +, ×):  abelian group under +, associative ×, distributive.
                   No inverses required for ×.   e.g. ℤ, polynomials, n×n matrices
FIELD  a commutative ring where every nonzero element has a × inverse.
                   You can add, subtract, multiply, AND divide.   e.g. ℚ, ℝ, ℂ, GF(p)
```

## Finite (Galois) fields

A field with finitely many elements — the bedrock of digital crypto and coding. They
exist exactly for prime-power sizes `pⁿ`.

```
GF(p)    = ℤₚ            integers mod a prime           (e.g. GF(7))
GF(2ⁿ)   = polynomials over GF(2), mod an irreducible polynomial

  GF(2⁸) — bytes as degree-7 polynomials — is AES's arithmetic.
  Addition = XOR;  multiplication = polynomial mult mod the AES polynomial.
```

Crucially **GF(2ⁿ) multiplication is not integer-mod arithmetic** — it is carry-less
(XOR-based) polynomial arithmetic, a common implementation trap.

## Homomorphisms

A **homomorphism** is a structure-preserving map: `f(a·b) = f(a)·f(b)`. It is the formal
statement that two structures "behave the same." The discrete-log and `gˣ` maps are
homomorphisms, and **homomorphic encryption** (compute on ciphertext without decrypting)
is exactly an operation that commutes through the encryption map — a frontier of
[privacy-preserving computation](../security/post_quantum_crypto.md).

## Where this shows up

- **Cryptography** — [public-key crypto](../security/encryption.md) lives in cyclic
  groups (discrete log, RSA); [elliptic-curve and post-quantum / lattice
  schemes](../security/post_quantum_crypto.md) are group/ring structures.
- **Coding & information theory** — Reed–Solomon and BCH error-correcting codes are
  built over GF(2ⁿ); ties into [information theory](information_theory.md).
- **Systems** — CRC checksums, hashing, and AES are finite-field arithmetic.
- **Foundations** — extends the modular arithmetic and number theory in
  [discrete math](discrete_math.md).

## Pitfalls

- **Dropping an axiom** — forgetting closure or inverse is the usual reason a set "isn't a
  group"; check all four.
- **GF(2ⁿ) ≠ integers mod 2ⁿ** — finite-field multiply is carry-less polynomial
  arithmetic, not ordinary modular multiplication.
- **Assuming commutativity** — matrix and permutation groups are non-abelian; `ab ≠ ba`.
- **Fields need prime-power size** — there is no field with 6 elements; `ℤₙ` is a field
  only when `n` is prime.
